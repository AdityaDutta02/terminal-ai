### 2. Credit Model

#### Pricing Constants (platform/lib/pricing.ts — new file)

```typescript
export const MODEL_TIER_CREDITS: Record<string, number> = {
  standard: 1,
  advanced: 4,
  premium: 6,
  'image-fast': 3,
  'image-pro': 93,
}

export const PLANS = {
  starter: { priceInr: 149, credits: 250, name: 'Starter' },
  creator: { priceInr: 299, credits: 650, name: 'Creator' },
  pro:     { priceInr: 599, credits: 1400, name: 'Pro' },
} as const

export const CREDIT_PACKS = {
  pack_100:  { credits: 100,  priceInr: 89  },
  pack_500:  { credits: 500,  priceInr: 399 },
  pack_2000: { credits: 2000, priceInr: 1499 },
} as const

export const WELCOME_CREDITS = 20  // granted after email verification
export const ANON_FREE_USES = 1    // per app per IP+cookie
export const CREATOR_FREE_CALLS = 50  // per app per month, testing only
```

#### Credit Math
- 1 credit = ₹0.60 to user
- 1 credit costs platform ≈ ₹0.09 in OpenRouter fees (standard tier)
- Platform gross margin on standard: (₹0.60 − ₹0.09) / ₹0.60 = 85%
- Creator gets 50% of credit value: ₹0.30 per credit
- Platform net (standard): ₹0.60 − ₹0.09 − ₹0.30 = ₹0.21 per credit (35% net)
- Superadmin channels: 100% to platform (no creator split)

#### Session Billing (replaces per-call billing)
- Credits deducted at session start (embed-token creation), not per API call
- `credits_per_session` = `MODEL_TIER_CREDITS[app.model_tier]`
- Free apps (`is_free = true`): deduct from creator's `creator_balance` not user's balance
- Anon usage: 1 free use, tracked via IP + cookie in `anonymous_usage` table

### 3. API Changes

#### 3a. `platform/app/api/embed-token/route.ts` — full rewrite

New decision tree:
1. Parse request: `{ appId, cookieId? }`, `Authorization` header (optional)
2. Fetch app: get `is_free`, `model_tier`, `credits_per_session`, `channel.is_superadmin_channel`, `channel.creator_balance`
3. **Anon path** (no session):
   - Check `anonymous_usage` for this `(appId, ip, cookieId)` — reject if already used
   - If free app: deduct from creator_balance, insert anon usage, issue token
   - If paid app: issue token with `credits_deducted = 0` (1 free use tracked)
   - Insert into `anonymous_usage`
4. **Auth path** (session present):
   - If free app: deduct from `creator_balance` (not user balance)
   - If paid app: check user balance ≥ `credits_per_session`, deduct atomically
   - Insert embed token with `credits_deducted` set
5. Issue JWT `{ userId|null, appId, sessionId, isAnon, creditsDeducted }`
6. Store token hash in `gateway.embed_tokens`

#### 3b. `platform/app/api/subscriptions/route.ts` — update plans

- Change plan amounts to ₹149/₹299/₹599
- `POST /api/subscriptions` — create Razorpay subscription, insert `user_subscriptions` record
- `DELETE /api/subscriptions` — cancel at period end
- `GET /api/subscriptions` — return current plan + next renewal + credits remaining

#### 3c. `platform/app/api/webhooks/razorpay/route.ts` — add handlers

New event handlers:
- `subscription.activated`: set status=active, `current_period_start/end`, grant `credits_per_month` via `grantCredits()`
- `subscription.charged`: advance period dates, grant next month's credits
- `subscription.cancelled`: set status=cancelled, set `current_period_end`
- `subscription.halted`: set status=paused
- `payment.captured` (existing): handle one-time credit pack purchase, call `grantCredits()`

#### 3d. `platform/app/api/embed-token/preview/route.ts` — new file

Anonymous preview endpoint (no auth required):
```
POST /api/embed-token/preview
Body: { appId, cookieId }
Headers: x-forwarded-for (Traefik injects)
```
- Enforce 1 free use per (appId, ip, cookieId)
- Free apps: deduct from creator_balance
- Paid apps: issue 1 free token (no credit charge to anyone)

#### 3e. `platform/app/api/credits/route.ts` — new endpoints

```
GET  /api/credits          — balance + ledger history (last 20 entries)
POST /api/credits/purchase — create Razorpay order for credit pack
```

#### 3f. `gateway/src/routes/proxy.ts` — remove credit deduction

- Remove `CREDITS_PER_REQUEST = 1` constant
- Remove `deductCredits` call
- Keep `logCall()` but add `creditsCharged` from token payload instead of hardcoded 1

#### 3g. `gateway/src/middleware/auth.ts` — simplify

- Remove `deductCredits` function entirely
- `embedTokenAuth` middleware: verify JWT + DB lookup (unchanged)
- Add token payload type: `isAnon: boolean`, `creditsDeducted: number`

### 5. Cron Infrastructure

#### 5a. `platform/app/api/cron/subscription-credits/route.ts`

```
GET /api/cron/subscription-credits
Header: Authorization: Bearer ${CRON_SECRET}
```

Daily job:
1. Find all active subscriptions where `credits_granted_at` is null OR `credits_granted_at < current_period_start`
2. For each: `grantCredits(userId, plan.credits_per_month, 'subscription_renewal')`
3. Update `credits_granted_at = NOW()`
4. Log to `cron_runs`

#### 5b. `platform/app/api/cron/creator-revenue/route.ts`

```
GET /api/cron/creator-revenue
```

Monthly job:
1. Sum `credit_ledger` entries where `reason = 'api_call'` and `app_id` is in creator's apps, for current month
2. Calculate creator share: 50% of credits spent (superadmin channels: 0%)
3. Insert to `creator_balance` (in-platform balance for future payout)
4. Log to `cron_runs`

#### 5c. `vercel.json` (or `platform/vercel.json`) cron entries

```json
{
  "crons": [
    { "path": "/api/cron/subscription-credits", "schedule": "0 1 * * *" },
    { "path": "/api/cron/creator-revenue", "schedule": "0 2 1 * *" }
  ]
}
```

Note: Platform is self-hosted (not Vercel), so crons must use a different mechanism. Use a system cron via `docker-compose` or BullMQ repeatable jobs in `deploy-manager`. Add `CRON_SECRET` env var and validate with `Authorization: Bearer` header on all cron routes.

### 1. Deploy-Manager Changes

#### 1a. `deploy-manager/src/queue/deploy-queue.ts` — config preflight

Before creating app in Coolify, validate:
1. `TERMINAL_AI_GATEWAY_URL` is set and not `"undefined"` — throw if missing
2. `TERMINAL_AI_APP_ID` is set and is a valid UUID — throw if invalid
3. Ping `${GATEWAY_URL}/health` — if not 200, fail job with `GATEWAY_UNREACHABLE`
4. Log all env vars that will be injected (redact secrets)

```typescript
async function preflightCheck(gatewayUrl: string, appId: string): Promise<void> {
  if (!gatewayUrl || gatewayUrl === 'undefined') {
    throw new Error('TERMINAL_AI_GATEWAY_URL is not set or is "undefined"')
  }
  if (!/^[0-9a-f-]{36}$/.test(appId)) {
    throw new Error(`Invalid TERMINAL_AI_APP_ID format: ${appId}`)
  }
  const res = await fetch(`${gatewayUrl}/health`, { signal: AbortSignal.timeout(5000) })
  if (!res.ok) throw new Error(`Gateway health check failed: ${res.status}`)
}
```

#### 1b. `deploy-manager/src/services/coolify.ts` — startup validation

After `setEnvVar()` calls but before `deployApplication()`:
1. Wait for deployment to reach `running` status (poll every 5s, timeout 3min)
2. Fetch app's public URL
3. `GET {appUrl}/health` — if 200 within 60s, mark deployment `success`
4. If health check fails 3 times: mark `failed`, store error in deployment record

#### 1c. Deploy-manager startup validation

On `deploy-manager` service boot (`src/index.ts`):
```typescript
const REQUIRED_ENV = [
  'GATEWAY_URL',
  'COOLIFY_URL',
  'COOLIFY_TOKEN',
  'COOLIFY_PROJECT_UUID',
  'COOLIFY_SERVER_UUID',
  'DATABASE_URL',
  'REDIS_URL',
]
for (const key of REQUIRED_ENV) {
  if (!process.env[key]) {
    console.error(`FATAL: Missing required env var: ${key}`)
    process.exit(1)
  }
}
```

```typescript
// terminal-ai-gateway.ts (generated)
async function callGateway(messages: Message[], retries = 1): Promise<Response> {
  const res = await fetch(`${GATEWAY_URL}/v1/chat/completions`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${getToken()}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({ model: config.model_tier, messages }),
  })

  if (res.status === 401 && retries > 0) {
    await refreshToken()
    return callGateway(messages, retries - 1)
  }

  return res
}
```
